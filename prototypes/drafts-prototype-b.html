Design A — Segmented Control ViewToggle-inspired

Draft tabs live inside a tinted trough (like the existing ViewToggle pattern). The active tab pops out with card-bg + border + shadow. Feels native to the design system — the most "expected" option.

Character: Familiar. Extends ViewToggle language. The tinted trough groups drafts visually.
  • The "Drafts" label adds clarity but takes horizontal space
  • × shows on hover/active for delete
  • Dashed + button disappears at max capacity
  • Tinted background separates it from the top bar without heavy borders
Design B — Floating Pills ThemePill-inspired

Individual pill-shaped cards with a colored dot indicator, like the theme pill. Active pill gets an accent border glow. Feels more like "objects" you switch between — each draft is a thing.

  • "+ New" as a dashed ghost pill — clear affordance
  • Slightly taller bar than Design A due to pill padding
Design C — Numbered Circles Minimal footprint

Ultra-compact numbered circles with an accent underline indicator. Barely there until you need it. Maximizes canvas real estate. The accent underline slides between active items.

Character: Disappears into the interface. Maximum canvas space.
  • Only 36px tall — thinnest option
  • Numbered circles are instantly scannable
  • Accent underline slides with CSS transition for active state
  • No explicit delete — would need long-press or context menu (trade-off)
  • "Draft 2 of 3" text adds context without labels on each circle
  • Risk: might be too minimal — could feel like pagination, not drafts
