Icon Placement Options

Where does the notification indicator live in the header? Includes a note on why standalone vs. integrated is a meaningful architectural choice — not just visual.

1 — Separate bell icon
Dedicated bell button next to the user button. Two separate dropdowns: bell opens notifications panel, avatar opens social menu (followers/logout). Clearest UX — each button has one job.

3
cristoph
States:

✓ Clear affordance — bell = notifications

✓ Large, easy tap target

✓ Scales well: notifications panel can grow independently

✗ Adds another button to an already-busy header

2 — Bell icon inside user button
Bell tucked into the left side of the user button. Problem: one button, one action. Clicking opens the user menu — but then what? You'd need to add a "Notifications" tab inside the existing social menu, which is already complex.

States:

✓ Saves header space

✗ Bell icon has no independent click target

✗ Forces notifications into the user menu (competing with social content)

✗ Dot-only: no count visible

3 — Dot on avatar, no bell
No bell at all — the avatar gets an accent dot when there are unread notifications. Ultra-minimal, but same problem as option 2: clicking the user button must choose between showing the social panel or notifications.

4 — Count badge on avatar
No bell, but a numbered badge overlays the avatar (mobile app-style). Shows count clearly. Same structural problem: one button must decide what it opens.
