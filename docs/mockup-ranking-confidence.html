3. Implementation Specification

Background

The ELO-based voting system produces unreliable rankings when voter turnout is low relative to submission count. This feature surfaces ranking confidence so users understand when results are solid vs. noisy. The confidence level is derived from the ratio voters / submissions.

LevelConditionMeaning
Highvoters >= submissionsRankings are reliable
Mediumvoters >= submissions × 0.5Rankings are roughly directional
Lowvoters < submissions × 0.5Rankings are noisy

Design decisions

  • Visual treatment: Colored pill badge (dot + label text on tinted background). Chosen over signal bars and text-only options for its scannability and familiarity (GitHub label pattern).
  • Pill label: Always the short word only — "High", "Medium", or "Low". Surrounding context provides the meaning ("Confidence" label in sidebar, "Confidence:" prefix in modal).
  • Section heading: "Ranking" (not "Voting") — this describes the ranking outcome and its reliability, not the act of voting.
  • Sidebar layout: Submissions, Voters, and Confidence displayed in a single flex-wrap row under the "Ranking" heading. All three stats sit on one line when space permits, and stack naturally when the sidebar is narrow.
  • Modal layout: Single line below the subtitle: {voters} voters · {submissions} submissions · Confidence: [pill] [?]. Subtle/secondary tone so it doesn't dampen the celebratory feel.
  • InfoTooltip: Same InfoTooltip component used elsewhere in the app. Appears next to the "Confidence" label (sidebar) and after the pill (modal). Same tooltip text in both locations.

Tooltip text

"Ranking confidence reflects how reliable the rankings are based on voter turnout. More voters relative to submissions means more accurate rankings. Low confidence means results may not reflect true community preference."

New CSS tokens (add to each theme × mode in src/index.css)

9 tokens per theme-mode combination (3 levels × 3 variants each):

TokenPurpose
--color-confidence-highDot color for high confidence
--color-confidence-high-bgPill background for high
--color-confidence-high-textPill text for high
--color-confidence-medDot color for medium confidence
--color-confidence-med-bgPill background for medium
--color-confidence-med-textPill text for medium
--color-confidence-lowDot color for low confidence
--color-confidence-low-bgPill background for low
--color-confidence-low-textPill text for low

Light modes use darker text on lighter tinted backgrounds. Dark modes use lighter text on darker tinted backgrounds. Exact values for all 6 theme-mode combinations are defined in the CSS at the top of this file — copy them directly.

New component

  • ConfidencePill — takes level: 'high' | 'medium' | 'low', renders a colored dot + short label. Used in both the sidebar card and the modal. Styled via the CSS tokens above, so it automatically adapts to all themes.

Data requirements

  • submission_count — total submissions for that day's challenge
  • voter_count — distinct voter_id from comparisons where winner_id IS NOT NULL
  • Confidence level is computed client-side from these two values using the ratio table above
